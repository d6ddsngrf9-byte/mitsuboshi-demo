## 1. ファイルとデプロイ

- **本体**：単一の自己完結HTML（`index.html`）。画像（ロゴ）も**base64でHTML内に埋め込み済み**なので、このファイル1枚で完結する。外部アセット依存なし。
- **サイズ**：約170KB（うちロゴbase64が約40KB×2＝ヒーロー用とフッター用で2回埋め込み）。
- **ローカル配置**：`~/mitsuboshi-demo/index.html`
- **デプロイ先**：Vercel（本番URL：**https://mitsuboshi-demo.vercel.app**）
  - team: `mitsuboshiart` / project: `mitsuboshi-demo`
- **デプロイ手順**（毎回これを実行）：
  ```bash
  cp ~/Downloads/mitsuboshi-art-hoshida-demo.html ~/mitsuboshi-demo/index.html   # ← Code運用ではこのcpは不要。直接 index.html を編集する
  cd ~/mitsuboshi-demo && vercel --prod
  ```
  ※Code運用に移ったら「Downloadsからcp」は廃止し、`~/mitsuboshi-demo/index.html` を直接編集 → `vercel --prod` の流れに統一するのがよい。
- **確認時の注意**：Vercelはキャッシュが残りやすい。反映確認は必ず **`Cmd+Shift+R`（スーパーリロード）**。

## 2. 技術構成（実装の要点）

- **単一HTMLファイル**（`<style>`インライン＋`<script>`インライン、フレームワークなし）。
- **日英バイリンガル**：`data-i18n="キー"` 属性 ＋ JSの `I18N` 辞書オブジェクト（`{ キー: { ja:..., en:... } }`）で全文切替。
  - リボン右端の「EN / 日本語」トグルボタンで切替。
  - 初回表示は**ブラウザ言語**（`navigator.language`）で自動判定：日本語以外なら英語表示。
  - i18nキーは現在 **約186個**。
  - **重要な落とし穴**：初回表示は「HTML本文にベタ書きされたテキスト」がそのまま出る。JS辞書は**トグル時にしか適用されない**。したがって **`<br>`や文言を変えるときは「HTML本文」と「JS辞書」の両方**を直さないと、初回表示と切替後で食い違う。（実際に voices-p の `<br>` がHTML本文に無くて折り返しが直らなかった事故あり）
- **配色**（CSS変数、`:root`）：
  - ベース白 × 水色（`--blue:#3F93B8` / `--blue-deep:#1F6885` / `--blue-soft:#EAF4F9`）× 金アクセント（`--gold:#E0A93A` / `--gold-deep:#9C701E`）。インク `--ink:#243B47`。
  - ロゴ本体が水色（#3F93B8系）。フッター（濃紺背景）ではロゴを `filter:brightness(0) invert(1)` で白抜き表示。
- **フォント**：Zen Old Mincho（見出し）＋ Zen Kaku Gothic New（本文）。Google Fonts。
- **セクション数**：14。上から
  1. リボン（DEMO注意書き＋言語トグル）
  2. ヒーロー（ロゴ＋「みんなの星で、銀河をつくる。」＋支援状況パネル※数値はサンプル）
  3. WHY（なぜやるのか）
  4. 代表メッセージ（小原さんの14年ストーリー）
  5. 三つの星（みつぼしの意味＝①生まれ育った人 ②移り住んだ人・つくる人 ③観に来る人）
  6. 2026年の挑戦（①子どもと銀河 ②招待作家7名との共同制作）
  7. 使いみち（助成金＋実カテゴリ）
  8. 支援コース（金額別グループ、下記参照）
  9. つくる人たち（縦番号レイアウト 01〜04）
  10. メンバーの声（作家名・写真あり/なし・同意取得済のみ）
  11. 開催情報（会期確定：2026/11/14土〜11/29日）
  12. 展示会場を提供くださる皆さま
  13. お支払いについて（Stripe＝信頼材料として1回だけ言及）
  14. FAQ →最後のCTA →SNSシェア →フッター（特商法＋ブランド）
- **JS機能**：言語切替 / ヒーローの星アニメ（SVG）/ 進捗バーアニメ / スクロールreveal / トースト通知（デモ時「本番ではStripe決済に進む」旨）/ SNSシェア（X・FB・IG・リンクコピー。IGはリンク直接シェア不可のため共有メニュー/コピーにフォールバック）。
- **注意**：`prefers-reduced-motion` 対応済み。`overflow-x:hidden` を html/body に設定済み（画面端の謎の帯対策）。

## 5. Stripe本番実装（次フェーズ・未着手）

デモが固まったら本番へ。**設計方針は確定済み**：

- **単一キャンペーン型**（マーケットプレイスではない）→ Stripe Connect不要、シンプル構成でよい。
- **Stripeアカウント・入金口座は必ず「星十色」名義**で作成（前田名義だと売上・税・返金責任が前田に集中するため厳禁）。
  - 池田泉州銀行の法人口座を入金先に指定可能。Stripe側の振込元は三井住友（Stripeインフラ側）なので、クライアントが三井住友口座を作る必要はない。
- **決済手段**：カード（数%）／コンビニ／銀行振込。
  - 銀行振込は自動消込・手数料約1.5%・返金手数料¥250・入金約4営業日。手動消込を避けたい（"疲れているオーナー"配慮）ためこれを推奨。
- **All-in方式**（目標未達でも開催・原則返金なし）→返金オペを避ける。特商法表記に反映済み。
- **購入型**（＝売上、寄付型ではない）。特商法表記必須（実装済み）。
- **鍵の扱い**：StripeシークレットキーはClaude Code側で `.env` に置き `.gitignore` で除外。チャット履歴には絶対に貼らない。→**これがCode移行の主要動機のひとつ**。
- 本番は進捗バー用のサーバーレス関数・Webhook受信・Checkout結線が必要になるため、CLIデプロイ（vercel等）前提。

## 7. 既知の落とし穴・トラブル履歴（同じ轍を踏まないため）

- **i18nの二重管理**：文言/`<br>`を変えたら「HTML本文」と「JS辞書」の**両方**を必ず更新（§2参照）。片方だけだと初回表示と切替後で食い違う。
- **フッター高さ揃え**：未解決。ブラウザ実視しながら要調整（§0-1）。
- **画面端の謎の帯**：`overflow-x:hidden`（html/body）で対処済み。トースト通知が画面外に隠れきってなかったのが一因で、`translateY(calc(100% + 80px))` で退避済み。
- **日本語の折り返し**（「のもと」が変な位置で折れる）：`max-width`が近接要因。`lead-p`は禁則（`line-break:strict`）を入れ、skyセクションは`max-width:none`。決定的には文節境界に`<br>`を入れて対応（ただしHTML本文側にも入れること）。
- **Vercelキャッシュ**：確認は必ず`Cmd+Shift+R`。「直ってない」の一部はキャッシュ起因の可能性。
- **ロゴ**：水色PNG。フッター濃紺背景では`filter:brightness(0) invert(1)`で白抜き。フッターロゴは`<a href="#">`でトップリンク済み。サイズはヒーロー192px（モバイル140px）／フッター180px。
